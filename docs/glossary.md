# Glossary

| Term | Meaning in AutomatosX |
|------|------------------------|
| **AutomatosX** | Sovereign AI orchestration & governance platform — the full system |
| **AX Studio** | General GenAI workspace (user endpoint). [Repo](https://github.com/defai-digital/ax-studio) |
| **AX Code** | AI coding & automation runtime (user endpoint). [Repo](https://github.com/defai-digital/ax-code) |
| **AX Engine** | Mac-native inference runtime (infrastructure). [Repo](https://github.com/defai-digital/ax-engine) |
| **AX Trust** | Governance layer — contracts, guardrails, audit, explainability (private enterprise product) |
| **AX Serving** | Enterprise orchestration — multi-node routing, heterogeneous compute (private enterprise product) |
| **AX Fabric** | Knowledge infrastructure — RAG, distillation, knowledge lifecycle (private enterprise product) |
| **Contract-first** | Execution and policy defined as explicit contracts for reproducibility |
| **Deterministic pipeline** | Same input → same output under contract-based execution |
| **Sovereign / offline** | Deployable without cloud dependency; air-gapped capable |
| **User endpoints** | Application entry points: Studio and Code |
| **Governance layer** | Trust & policy enforcement between users and infrastructure |
| **AI infrastructure** | Serving, Fabric, Engine |
| **Heterogeneous compute** | Mix of Mac, edge (e.g. Jetson Thor), and future cluster nodes |
| **OpenAI-compatible** | HTTP API shape used so Studio/Code can call Engine (or other runtimes) |
| **MCP** | Model Context Protocol — tool integration for agents and workspaces |
| **DEFAI** | DEFAI Digital — organization that builds AutomatosX |

Product-specific terms are defined in each product’s own documentation.
